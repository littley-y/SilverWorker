## 2. 주요 변경 사항

### 2.1 `JobRepository` Firestore 연동 (`lib/repositories/job_repository.dart`)

```dart
Future<List<JobModel>> fetchJobs(JobFilter filter) async {
  Query query = FirebaseFirestore.instance
      .collection('jobs')
      .where('isActive', isEqualTo: true)
      .where('deadline', isGreaterThan: Timestamp.now());

  if (filter.locationCode != null) {
    query = query.where('locationCode', isEqualTo: filter.locationCode);
  }
  if (filter.jobCategory != null) {
    query = query.where('jobCategory', isEqualTo: filter.jobCategory);
  }

  query = query.orderBy('deadline').limit(50);
  final snapshot = await query.get();
  return snapshot.docs
      .map((doc) => JobModel.fromJson(doc.data()! as Map<String, dynamic>))
      .toList();
}
```

- `fetchJobs()`: 활성 공고를 마감일 순으로 50개까지 조회. `locationCode`, `jobCategory` 선택적 필터 지원.
- `fetchJobById()`: 단일 공고 상세 조회 (spec_05 대비).
- `as Map<String, dynamic>` 캐스트: `cloud_firestore` 5.x의 `QueryDocumentSnapshot.data()` 반환 타입이 `Object?`로 추론되는 이슈 해결.

## 5. 알려진 제한 사항

1. **고용24 API 미연동**: 경로 A(Mock)로 구현되어 실제 API 데이터가 아님. 향후 API 키 발급 시 경로 B(Cloud Functions 프록시)로 전환 가능.
2. **Firestore 읽기 권한**: `/jobs` 컬렉션이 `allow read: if true;`로 설정되어 인증 없이 읽기 가능. MVP 데모 단계에서는 의도된 동작이나, 프로덕션에서는 Rate Limiting 등 추가 보안 고려 필요.
3. **Mock 데이터 경직성**: `seed_jobs.py`가 랜덤 생성이므로 실행 시마다 다른 데이터가 생성됨. 재현 가능한 테스트를 위해 시드 고정이 필요할 수 있음.
