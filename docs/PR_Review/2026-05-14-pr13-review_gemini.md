## 2. 주요 리뷰 포인트 분석

### 2.1 아키텍처 (Architecture)
*   **`visibleJobListProvider` 설계**:
    `authStateProvider`와 `myApplicationsProvider`를 조합하여 지원 내역을 필터링하는 로직이 매우 깔끔합니다. 인증 상태가 바뀌거나 새로운 지원/취소가 발생할 때 자동으로 리렌더링(rebuild)되도록 Riverpod의 강점을 잘 살렸습니다. 특히 `try-catch` 블록에서 예외 발생 시 빈 목록이 아닌 전체 공고(`jobs`)를 반환하도록 한 점(Graceful Degradation)은 훌륭한 안전망(Fail-safe) 설계입니다.
*   **Firestore 쿼리 개선 (`fetchApplications`)**:
    `.where('status', isNotEqualTo: 'cancelled')`를 추가한 것은 앱 내 데이터 정합성을 높입니다. 복합 인덱스가 이미 배포되어 있다면 성능 이슈 없이 완벽하게 작동할 것입니다.
*   **PhoneInputScreen E.164 포맷 수정**:
    한국 휴대전화 번호에서 앞자리 `0`을 제거하지 않고 바로 `+82`를 붙이는 로직으로 수정한 것은, 경우에 따라 Firebase Auth SMS 발송 라우팅에서 필요한 국내 통신사 표준을 맞추기 위한 적절한 롤백/수정으로 보입니다.

### 2.2 UI/UX
*   **홈 화면 지원 공고 필터링**:
    시니어 사용자가 "내가 여기에 지원했었나?" 헷갈리지 않게 아예 목록에서 지워주는 것은 최상의 사용자 경험입니다. 지원 취소 시 다시 목록에 나타나므로 플로우상 문제도 없습니다.
*   **플로팅 카드 (지원 취소 피드백)**:
    지원 취소 직후 화면 하단에 플로팅 카드를 3초간 띄워 명확한 시각적 피드백을 제공한 점이 좋습니다. 일반적인 스낵바(SnackBar)보다 가시성이 높습니다.
*   **설정 및 마이페이지 정리**:
    기능이 제거된 글자 크기 조절 섹션을 UI에서 아예 걷어냄으로써 화면을 심플하게 유지했습니다. 은일이 마스코트를 홈 화면(배너)에만 집중 배치하여 너무 산만해 보일 수 있는 위험을 줄였습니다.

### 2.3 코드 품질 (Code Quality)
*   **테스트 일관성**:
    제거된 UI에 맞게 `settings_screen_test.dart`의 `expect` 구문들을 꼼꼼하게 정리하여 CI가 통과하도록 유지했습니다.
*   **Mock 데이터 보강 (`seed_jobs.py`)**:
    `walkingMinutes` (도보 N분) 필드를 추가하고 `JobCard`에 `GrayChip`으로 연결한 것은 위치 기반 매칭이라는 앱의 본질에 다가가는 좋은 스펙업입니다. 목업 데이터에 실제 기업명을 사용한 것도 화면의 현실감을 크게 높여줍니다.
