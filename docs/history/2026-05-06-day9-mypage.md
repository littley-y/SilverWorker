## 구현 내용

### 1. MypageScreen (`lib/screens/mypage/my_page_screen.dart`)
- **프로필 요약 카드**: 이름(22pt Bold), 거주 지역(18pt), 경력 소개(16pt, 최대 2줄 + 말줄임), 지원 횟수 뱃지("총 N건 지원")
- **메뉴 리스트**: 지원 내역 → ApplicationListScreen, 찜한 공고(추후), 알림 설정(추후)
- **로그아웃 버튼**: 하단 텍스트 버튼(빨간색). 탭 시 확인 다이얼로그("로그아웃 하시겠습니까?") → 확인 시 `FirebaseAuth.instance.signOut()`
- **데이터 로드**: `userProfileProvider`, `myApplicationsProvider` 사용
- **상태 처리**: loading, error, null profile(→프로필 등록 화면 리다이렉트) 모두 처리

### 2. ApplicationListScreen + ApplicationCard (`lib/screens/mypage/application_list_screen.dart`)
- **ApplicationListScreen**: AppBar("지원 내역"), `myApplicationsProvider` 기반 ListView, 빈 상태("아직 지원한 공고가 없습니다")
- **ApplicationCard**: 공고 제목(18pt Bold), 회사명(16pt 회색), 지원 일시(14pt 회색, MM월 DD일), 상태 배지(우측)
- **상태 배지 색상**:
  - `submitted` → 파랑 (`AppColors.statusSubmitted`)
  - `reviewing` → 주황 (`AppColors.statusReviewing`)
  - `accepted` → 초록 (`AppColors.statusAccepted`)
  - `rejected` → 빨강 (`AppColors.statusRejected`)
  - `cancelled` → 회색 (`AppColors.statusCancelled`)
- 정렬: `submittedAt` 내림차순 (repository 레벨에서 처리)

### 3. Router 업데이트 (`lib/router/app_router.dart`)
- 신규 라우트: `/mypage` → MyPageScreen, `/mypage/applications` → ApplicationListScreen

### 4. 테스트 (`test/widgets/`)
- `my_page_screen_test.dart` (6건): 프로필 표시, 지원 횟수, 메뉴, 로그아웃 버튼, 다이얼로그, 로딩 상태
- `application_list_screen_test.dart` (5건): 빈 상태, 카드 렌더링, 5종 상태 배지 색상, 에러 상태, 로딩 상태
