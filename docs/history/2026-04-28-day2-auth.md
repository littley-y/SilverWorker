## 3. Auth Flow

```
App Launch
  ↓
GoRouter checks FirebaseAuth.currentUser
  ├─ Not logged in → /auth/phone
  │    ↓
  │  PhoneInputScreen → verifyPhoneNumber()
  │    ↓
  │  /auth/otp ← verificationId
  │    ↓
  │  OtpInputScreen → signInWithCredential()
  │    ↓
  │  / ← auto-redirect on auth state change
  │    ↓
  │  MainScreen checks Firestore profile
  │      ├─ No profile → /auth/profile
  │      └─ Has profile → JobListScreen
  │
  └─ Logged in + has profile → / (MainScreen → JobListScreen)
```

## 4. UI Specifications

### PhoneInputScreen
- Title: "휴전폰 번호로 시작하세요" (24pt Bold)
- Input: `+82` prefix (fixed) + 10–11 digit number
- Button: "인증번호 받기" (56dp height, primary color)
- Error: red border + "올바른 번호를 입력하세요"

### OtpInputScreen
- Title: "인증번호 6자리를 입력하세요" (22pt Bold)
- 6 separate input fields (56dp × 64dp each)
- Resend button: 60-second countdown, then enabled
- Verify button: disabled until all 6 digits entered

### ProfileSetupScreen
- Name: required, max 20 chars
- Address: 2-step dropdown (sido → sigungu)
- Career summary: optional, max 500 chars with live counter
- Submit: "시작하기" (disabled until name + address selected)

## 5. Firestore Schema

`createProfile` writes to `/users/{uid}`:

```dart
{
  'userId': uid,
  'phoneNumber': phoneNumber,
  'name': name,
  'address': {'sido': sido, 'sigungu': sigungu},
  'careerSummary': careerSummary,
  'gender': '',
  'physicalConditions': [],
  'preferredJobTypes': [],
  'preferredLocations': [],
  'isPushEnabled': true,
  'createdAt': FieldValue.serverTimestamp(),
  'updatedAt': FieldValue.serverTimestamp(),
}
```

Aligned with `docs/planning/overview/04_db_schema.md` §2.1.
